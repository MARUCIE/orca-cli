差距 3：安全沙箱

来源: Codex CLI（Seatbelt, bwrap, Landlock）| P1 高优

Codex CLI 使用操作系统级别的沙箱隔离（macOS Seatbelt、Linux bubblewrap+Landlock），硬件级不可绕过。Orca 当前仅有 shellEscape 字符转义和 safe/yolo 二档权限切换，不足以在不可信环境运行。

差距 4：技能系统（Skills）

来源: AI-Fleet（503 技能，28 组）+ Hermes（123 技能）| P1 高优

AI-Fleet 拥有 4 种执行模式（蜂群/管道/循环/顺序）的技能编排引擎，KiloCode 有行为配置系统（Mode System）。Orca 当前仅有基础的上下文文件加载，缺乏技能执行引擎和注册表。

差距 5：平台网关

来源: Hermes（15+ 渠道）| P2 中优

Hermes 通过单一网关架构接入 Telegram、Discord、Slack、WhatsApp 等 15+ 平台。Orca 当前仅有 CLI 和基础 HTTP serve 两个入口，无法触达终端以外的用户。

差距 6：持久化记忆与跨会话学习

来源: OpenClaw + AI-Fleet（DNA 胶囊系统）| P2 中优

AI-Fleet 的 DNA 胶囊系统实现跨会话知识继承——将修复模式编码为可复用胶囊，新会话自动匹配症状并注入修复方案。OpenClaw 有自我改进 Agent 模式。Orca 当前仅有 session save/load，无学习能力。

2026-04-13 | 用户 /council 实际运行失败分析 → P0 级核心差异化修复

根因分析：用户执行 /council 审核报告。预期：Claude Opus 4.6 + GPT 5.4 + Gemini 3.1 Pro 三模型并行分析。实际：所有模型返回 "No provider endpoint found" — findAggregator() 不认识 Poe 是聚合器。

层级位置触发条件行为

L2 内部预算 streamChat 每轮 API 调用前 messages 估算 > 75% window 截断 旧 tool_result 到 150 chars
L3 外部 guard runProxyTurn PostToolUse 后 history > 60% budget compact 丢弃旧消息

根因：之前只有 L3+L4，但 L1/L2 缺失导致 streamChat 内部 messages 膨胀到 320K 时外部 guard 完全不可见。v0.7.1 修复后四层全覆盖。

v0.8.0 新增：Mission Mode + Task Planner + Nuclear Compact

v0.8.0 是架构跃迁版本：从"单任务 Agent"升级为"多目标并发编排器"。核心新增 3 个模块：Mission Mode（Droid 三层架构）、Task Planner（自动拆分+并发执行+可视化清单）、Nuclear Compact（核弹级上下文恢复）。同时完成 UI/UX 系统性打磨。

Mission Mode（src/mission/）

灵感来源：Factory.ai Droid Missions 架构。Orchestrator / Worker / Validator 三层分离，验证契约优先。

组件职责实现方式
Orchestrator分解目标为 Milestone → Feature，生成 ValidationContractchatOnce 调用 LLM，输出结构化 JSON blocks
Worker实现单个 Feature（每次获得全新上下文）spawnSubAgent fork 隔离进程，DELEGATE_TOOLS
Validator独立验证 — 对照契约，不对照实现evaluateCriterion: command / regex / file_exists

关键设计：ValidationContract 在 Feature 分解之前生成（防止 implementation bias）；每个 Worker 获得独立上下文（防止 context rot）；Validator 不看实现代码（防止 self-evaluation bias）。

入口：/mission <goal>（REPL 交互式）/ orca run --mission <goal>（headless）

Task Planner（src/planner/）

核心 SOTA 差异化：用户输入一段包含多个任务的提示词，系统自动拆分为主线（顺序）+ 支线（并发）任务清单。

模块功能亮点
decomposer提示词 → TaskPlan 拆分LLM 分解 + 启发式 fallback（编号列表 / bullet / 中文分号 / 英文连词）
tracker可视化待办清单 + 状态生命周期Box-drawing 清单，每完成一个回到清单标记 ✓
executor并发执行引擎main 顺序 + side 并发（spawnSubAgent，最大 3 并发）

自动检测：isMultiTaskPrompt() 在 REPL 输入时自动识别多任务提示词，提示用户 /plan。

入口：/plan <tasks>（REPL）/ orca run --plan <tasks>（headless）
