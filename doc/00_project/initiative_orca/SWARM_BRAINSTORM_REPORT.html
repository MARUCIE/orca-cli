概要结论

Orca CLI v0.3.0 拥有扎实基础（41 个工具、10 个钩子事件、多模型 council/race/pipeline、493 个测试用例）。但对标 SOTA 级编码代理（Claude Code、Codex CLI、Amp CLI、KiloCode、Hermes），Orca 在 6 个关键维度存在结构性差距，阻碍其成为真正的 SOTA 级 CLI 工具。

好消息是：AI-Fleet 已经对大部分差距有成熟实现，移植路径清晰，风险可控。

6 个关键差距维度

差距 1：线束工程层（Harness）

来源: AI-Fleet core/harness/（18 模块，~196K Python）| P0 紧急

AI-Fleet 的 18 模块线束层是其核心竞争力。包含预完成门禁、循环检测器、上下文监控器、错误分类器等。这是弥合"模型能力"与"代理交付"之间 20 分差距的关键层。

Phase 1 进展：4 个 P0 模块已移植为 TypeScript 并接入 Agent 循环，29 个新测试全部通过。

模块功能状态
verification-gate预完成质量门禁（lint/typecheck/test）已完成
loop-detectorTw93 规则：2 次失败转向，3 次上报已完成
context-monitor4 级风险：绿/黄/橙/红已完成
error-classifier9 类错误 + 恢复建议已完成
trace-collector结构化执行追踪P1 待实现
quality-scorer5 维度 A-F 评分P1 待实现
knowledge-compounder修复模式 -> DNA 胶囊P2 待实现

差距 2：代理隔离与多代理协作

来源: Claude Code（Agent Teams, Worktree）| P0 紧急

Claude Code 的 Agent Teams 通过 git worktree 实现代理间代码隔离，每个 worker 在独立分支上工作，互不干扰。Orca 当前的 spawn_agent 和 delegate_task 均为空壳实现。

功能Claude CodeOrca差距
子代理生成Haiku 做探索，Opus 做推理异步桩需要真正的进程隔离
Worktree 隔离每个代理独立 git 分支无完全缺失
Agent 团队Leader + N Workers 编排无完全缺失
模型分层委托廉价模型做搜索，高级模型做推理单一模型完全缺失

差距 3：安全沙箱

来源: Codex CLI（Seatbelt, bwrap, Landlock）| P1 高优

Codex CLI 使用操作系统级别的沙箱隔离（macOS Seatbelt、Linux bubblewrap+Landlock），硬件级不可绕过。Orca 当前仅有 shellEscape 字符转义和 safe/yolo 二档权限切换，不足以在不可信环境运行。

差距 4：技能系统（Skills）

来源: AI-Fleet（503 技能，28 组）+ Hermes（123 技能）| P1 高优

AI-Fleet 拥有 4 种执行模式（蜂群/管道/循环/顺序）的技能编排引擎，KiloCode 有行为配置系统（Mode System）。Orca 当前仅有基础的上下文文件加载，缺乏技能执行引擎和注册表。

差距 5：平台网关

来源: Hermes（15+ 渠道）| P2 中优

Hermes 通过单一网关架构接入 Telegram、Discord、Slack、WhatsApp 等 15+ 平台。Orca 当前仅有 CLI 和基础 HTTP serve 两个入口，无法触达终端以外的用户。

差距 6：持久化记忆与跨会话学习

来源: OpenClaw + AI-Fleet（DNA 胶囊系统）| P2 中优

AI-Fleet 的 DNA 胶囊系统实现跨会话知识继承——将修复模式编码为可复用胶囊，新会话自动匹配症状并注入修复方案。OpenClaw 有自我改进 Agent 模式。Orca 当前仅有 session save/load，无学习能力。
