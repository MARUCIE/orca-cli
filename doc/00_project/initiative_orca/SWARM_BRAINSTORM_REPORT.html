6 Critical Gap Dimensions

Gap 1: Harness Engineering Layer

Source: AI-Fleet core/harness/ (18 modules, ~196K Python) | P0 CRITICAL

AI-Fleet 的 18 模块 harness 层是其核心竞争力。包含 verification gate（预完成门禁）、loop detector（循环检测）、context monitor（上下文监控）、error classifier（错误分类）等。这是弥合"模型能力"与"代理交付"之间 20 分差距的关键层。

Phase 1 已完成：4 个 P0 模块已移植为 TypeScript（loop-detector, context-monitor, error-classifier, verification-gate），29 个新测试全部通过。

Gap 2: Agent Isolation & Multi-Agent

Source: Claude Code (Agent Teams, Worktree) | P0 CRITICAL

Claude Code 的 Agent Teams 通过 git worktree 实现代理间的代码隔离，每个 worker 在独立分支上工作。Orca 当前 spawn_agent 和 delegate_task 均为空壳。

Gap 3: Security Sandbox

Source: Codex CLI (Seatbelt, bwrap, Landlock) | P1 HIGH

Codex CLI 使用 OS 级别的沙箱隔离（macOS Seatbelt, Linux bubblewrap+Landlock），硬件级别不可绕过。Orca 当前仅有 shellEscape + binary safe/yolo 两档权限。

Gap 4: Skills System

Source: AI-Fleet (503 skills, 28 groups) + Hermes (123 skills) | P1 HIGH

AI-Fleet 拥有 4 种执行模式（swarm/pipeline/loop/sequential）的技能编排引擎，KiloCode 有 Mode System。Orca 当前仅有基础的 context loading，没有技能执行引擎。

Gap 5: Platform Gateway

Source: Hermes (15+ channels) | P2 MEDIUM

Hermes 通过单一 Gateway 接入 Telegram、Discord、Slack、WhatsApp 等 15+ 平台。Orca 当前仅有 CLI + HTTP serve 两个入口。

Gap 6: Persistent Memory & Learning

Source: OpenClaw + AI-Fleet (DNA Capsules) | P2 MEDIUM

AI-Fleet 的 DNA 胶囊系统实现跨会话知识继承——将修复模式编码为可复用的胶囊，新会话自动匹配症状并注入修复方案。Orca 当前仅有 session save/load。
