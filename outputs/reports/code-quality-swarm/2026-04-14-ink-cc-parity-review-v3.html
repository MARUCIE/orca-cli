Executive Summary

三轮蜂群深度审查覆盖了 CC（Claude Code）源码的渲染管线（AlternateScreen / FullscreenLayout / ScrollBox / Resize）、输入系统（Cursor 类 / 粘贴处理 / Focus 模型 / 提交流程）、视觉 UX（主题系统 / Spinner / 工具调用状态 / 键盘提示）三大核心层。

结论：Orca 已完成 6/17 关键架构差距修复（AlternateScreen + 底部钉定 + stdin 捕获 + 输入缓冲 + 文本可见性 + React 状态修复），解决了全部用户反馈的 UX 阻塞问题。剩余 11 项差距中，3 项 P0（ScrollBox / 粘贴处理 / 动态 Resize）直接影响长对话可用性，4 项 P1（Cursor 模型 / 主题扩展 / Focus 精细控制 / 鼠标追踪）影响编辑体验与专业性，4 项 P2（kill/yank / 图片粘贴 / Spinner 升级 / 工具调用分级错误渲染）为体验增强。

Orca 的 6 项独有优势（SIGCONT 恢复 / 双行 StatusBar / 上下文感知键盘提示 / Sparkline / DiffPreview / OSC 8 链接）已超越 CC 对应实现，构成产品差异化。

三、Orca 独有优势（超越 CC）

以下 6 项是 Orca 已有但 CC 没有的功能，构成产品差异化亮点。

1. SIGCONT Resume Handler

Orca 在 AlternateScreen 中监听 SIGCONT 信号，当用户 Ctrl+Z 挂起 → fg 恢复时，自动重新进入备用屏幕缓冲区。CC 没有这个处理，恢复后显示混乱。

AlternateScreen.tsx:42-48

2. 双行 StatusBar 布局

Line 1：model / permMode / branch / cost / tok/s。Line 2：unicode 进度条 + 百分比 + turns + sparkline。CC 单行合并所有信息，窄终端下可读性差。Orca 的双行设计在窄终端更清晰。

StatusBar.tsx

3. Token Sparkline 可视化

Unicode braille 字符（▁▂▃▄▅▆▇█）逐轮渲染 token 使用趋势。CC 无此功能。直观展示对话 token 消耗走势。

StatusBar.tsx sparkline 渲染 + chat.ts SessionStats.turnTokens[]

4. 上下文感知键盘提示

Footer 组件根据 3 种状态（generating / inputActive / idle）动态切换快捷键提示。CC 的 KeyboardShortcutHint 较为静态。Orca 的提示更贴合当前操作语境。

Footer.tsx:24-40

5. Inline DiffPreview 组件

文件编辑权限请求时，内联渲染彩色逐行 diff（绿色新增 / 红色删除 / 灰色上下文）。CC 的 FileEditToolDiff 是事后渲染，Orca 是权限审批前预览。

DiffPreview.tsx + PermissionRequest.diff

6. OSC 8 文件路径超链接

支持终端的文件路径渲染为 Cmd+Click 可点击链接（iTerm2 / Warp / kitty）。CC 无此功能。

FileLink.tsx

四、已修复的关键 Bug（用户反馈驱动）

Bug 1: 空状态巨大空白（用户截图 x3）

根因：height={rows} + 空内容 → 输入框推到终端最底部

修复：AlternateScreen 深色背景填充 + content flexGrow=1 + bottom flexShrink=0

Commit：9372ae6

Bug 2: 键盘输入落到 shell（用户截图 x2）

根因：useInput({ isActive: false }) 在 prompt_ready 之前不进入 raw mode

修复：isActive: !permissionBlocked 始终捕获 stdin + 输入缓冲机制

Commit：73b9bf4

Bug 3: React "Cannot update App while rendering InputArea"

根因：setValue wrapper 在 render 期间调用 onChange → parent setInputValue

修复：onChange 改用 useEffect + useRef，在 commit phase 后通知 parent

Commit：73b9bf4

Bug 4: 响应文本提交后消失

根因：<Static> 与 fullscreen layout 不兼容（overflow:hidden 裁剪）+ textBuffer flush 竞态

修复：替换 <Static> 为 blocks.map() + turn_summary/prompt_ready 时合并 textBuffer

Commit：ccb607c

Bug 5: 内容挤到底部

根因：justifyContent="flex-end" 把 banner 和空状态引导也推到底部

修复：移除 flex-end，内容从顶部自然流动

Commit：73b9bf4

五、PDCA 执行计划

基于三轮审查结果，按优先级排列实施计划。P0 为基础可用性阻塞，P1 为专业体验，P2 为锦上添花。

P0 — 基础可用性（阻塞长对话使用）

项目做什么验证方式
ScrollBox 组件 实现自定义 ScrollBox：
- scrollToElement(el, offset) 定位到指定元素
- scrollBy(delta) 增量滚动
- isSticky() 自动跟随底部
- 键盘：PageUp/PageDown/Home/End
- 替换 App.tsx content area 的 overflow="hidden" 		50+ 条消息对话中可以回滚查看历史
粘贴处理 实现 usePasteHandler hook：
- 检测 bracketed paste mode（\x1b[200~ / \x1b[201~）
- 粘贴期间抑制 Enter 提交
- isPasting 状态追踪
- 多行粘贴正确插入 		粘贴含换行的代码片段不会意外提交
动态 Resize 实现 useTerminalSize hook：
- 监听 stdout.on('resize')
- 通过 Context 传播 rows/cols
- AlternateScreen / StatusBar / InputArea / Footer 响应式重排 		终端窗口缩放后布局正确适应

P1 — 专业体验（编辑效率与视觉精细度）

项目做什么验证方式
Cursor 模型 提取独立 Cursor 类：
- prevWord() / nextWord()（Ctrl+Left/Right，Option+Left/Right）
- deleteWordBefore()（Ctrl+W）
- deleteToLineStart()（Ctrl+U 升级）
- wrap-aware 行号计算 		Ctrl+W 删除一个词、Option+Arrow 跳词
Theme 扩展 扩展到 30+ 语义色 + dark/light 双模式：
- 从 5 accent 色扩展到分角色语义色（model、tool、error、success、dim、accent、prompt 等）
- 自动检测终端 dark/light mode
- ANSI 256 / TrueColor 双通道 		不同终端主题下视觉一致
Focus 精细控制 重构输入焦点为 multi-source：
- permissionBlocked + pickerActive + modalActive + searchActive
- 独立 showCursor prop（焦点失去但保持光标可见） 		CommandPicker / PermissionPrompt / 其他模态各自独立控制焦点
鼠标追踪 启用 SGR 鼠标模式：
- \x1b[?1003h（所有鼠标事件）
- Wheel 事件 → ScrollBox.scrollBy()
- 退出时 \x1b[?1003l 清理 		鼠标滚轮可以滚动内容
